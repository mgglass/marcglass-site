Setup & Publishing Guide

1Organize your files on your computer

▾

Create a folder on your computer for your site. Inside it, you need:

  • index.html — your main site file (downloaded from Claude)
  • photo-admin.html — this tool
  • photos/ — a folder where all your images go

When you reference a photo in the code, use the path photos/filename.jpg — for example: src="photos/sunset-portland.jpg"

Photo tips: Rename your files with simple, lowercase names (no spaces — use hyphens instead). Example: maine-harbor-2024.jpg not My Photo (1).jpg.

2Create a free GitHub account and repository

▾
  1. Go to github.com and create a free account if you don't have one.
  2. Click the green New button to create a new repository.
  3. Name it marcglass.com (or any name you like).
  4. Set it to Public and check Add a README file.
  5. Click Create repository.

3Upload your files to GitHub

▾

The easiest way — no command line needed:

  1. Open your repository on GitHub.
  2. Click Add file → Upload files.
  3. Drag your index.html and photo-admin.html files into the upload area.
  4. Click Commit changes.
  5. Then create the photos/ folder: click Add file → Create new file, type photos/.gitkeep in the filename box, and commit. This creates the folder.
  6. Go into the photos/ folder and upload your images the same way.

You can also use the free GitHub Desktop app for a more drag-and-drop experience: desktop.github.com

4Turn on GitHub Pages (free hosting)

▾
  1. In your GitHub repository, click Settings (top menu).
  2. Scroll down to Pages in the left sidebar.
  3. Under Source, select Deploy from a branch.
  4. Choose main branch and / (root) folder.
  5. Click Save.
  6. Wait 2–3 minutes. Your site will be live at yourusername.github.io/repositoryname.

5Connect your marcglass.com domain

▾

You'll do this in two places — GitHub and your domain registrar (GoDaddy, Namecheap, etc.).

In GitHub Pages settings:

  1. Under Pages → Custom domain, type marcglass.com and click Save.
  2. Check the Enforce HTTPS box once it appears.

At your domain registrar:

  1. Find the DNS settings for marcglass.com.
  2. Add these 4 A records pointing to GitHub's servers:
    185.199.108.153
    185.199.109.153
    185.199.110.153
    185.199.111.153
  3. Add a CNAME record: www pointing to yourusername.github.io

DNS changes take 10 minutes to 48 hours to fully propagate. Once done, marcglass.com will load your GitHub Pages site.

6Updating your site after launch

▾

Any time you want to update text or add/change photos:

  1. Edit your index.html file locally (or use GitHub's built-in editor — click the file, then the pencil icon).
  2. Upload new photos to the photos/ folder via Add file → Upload files.
  3. Use this Photo Manager tool to generate the HTML code for new photos, then paste it into the right spot in index.html.
  4. Commit your changes — GitHub Pages automatically rebuilds and publishes within about 60 seconds.

7Set up the contact form (Formspree — free)

▾
  1. Go to formspree.io and create a free account.
  2. Click New Form, give it a name (e.g. "Marc Glass Contact"), and enter your email address.
  3. Copy the form ID — it looks like xrgpblkn.
  4. In index.html, find the line:
    action="https://formspree.io/f/YOUR_FORM_ID"
  5. Replace YOUR_FORM_ID with your actual ID.
  6. That's it! Form submissions go directly to your email, free forever up to 50/month.
